In this blog I want to describe how I handle my projects typography.
A clear example of it is my portfolio which is open source and you can check it out [here](https://github.com/miladMAPS/portfolio).

## Why to use a global typography?

Using global typography for sizes of texts increases the readability and maintainability of code and helps you design as clearly and efficiently as possible.

- Benefits
  - Increasing maintainability
    - We can easily change the scale of our fonts at any time
  - Increasing code readability
  - Ability to have multiple typographies for different screen dimensions
  - ...

## Implementing

### 1. Defining Breakpoints

First, we need to set our breakpoint configs

```javascript
const breakpoints = {
  xs: 414,
  sm: 576,
  md: 768,
  lg: 992,
  xl: 1200,
};
```

### 2. Defining typography variants

Then we need to define our typography variant in different breakpoints.
In the below example the key `default` will be our typography in screens with _min-width_ of **breakpoints.lg**.

```javascript
const typographyVariants = {
  default: {
    "h-lg": { fontSize: 24, fontWeight: "normal" },
    "h-lg-b": { fontSize: 24, fontWeight: "bold" },
    "h-md": { fontSize: 20, fontWeight: "normal" },
    "h-md-b": { fontSize: 20, fontWeight: "bold" },
    "h-sm": { fontSize: 18, fontWeight: "normal" },
    "h-sm-b": { fontSize: 18, fontWeight: "bold" },
    lg: { fontSize: 16, fontWeight: "normal" },
    "lg-b": { fontSize: 16, fontWeight: "bold" },
    md: { fontSize: 13, fontWeight: "normal" },
    "md-b": { fontSize: 13, fontWeight: "bold" },
    sm: { fontSize: 11, fontWeight: "normal" },
    "sm-b": { fontSize: 11, fontWeight: "bold" },
    xsm: { fontSize: 10, fontWeight: "normal" },
    "xsm-b": { fontSize: 10, fontWeight: "bold" },
    xxsm: { fontSize: 9, fontWeight: "normal" },
    "xxsm-b": { fontSize: 9, fontWeight: "bold" },
  },
  [breakpoints.lg]: {
    // ...
  },
};
```

### 3. Adding values as global CSS variables.

After that, we need to add those typography variants as global CSS variables to use them everywhere.
We can reach that with `createGlobalStyle` method from [styled-components](https://styled-components.com/).

This adds default typography variant variables under **:root** and adds other variants under corresponding media queries to override defaults.

```javascript
// adding values as CSS variables like : var(--font-size-sm) and var(--font-weight-sm)
import { createGlobalStyle } from "styled-components";
import { typographyVariants } from "@stylesheets";
import __reduce from "lodash/reduce";
\
export const GlobalTypographies = createGlobalStyle((props) => {
  return __reduce(
    typographyVariants,
    (accumulatedStyles, typography, breakpoint) => {
      const variables = {
        ":root": __reduce(
          typography,
          (acc, { fontSize, fontWeight }, key) => ({
            ...acc,
            ['--font-size-' + key]: fontSize,
            ['--font-weight-' + key]: String(fontWeight),
          }),
          {}
        ),
      };
      return {
        ...(breakpoint === "default"
          ? variables
          : { ["@media (min-width:" + breakpoint + "px)"]: variables }),
        ...accumulatedStyles,
      };
    },
    {}
  );
});
```

<small>
  ** We need to import <strong>GlobalColors</strong> component into our most
  top-level component like App.js in react or _app.js in Next.js.
</small>

<br />

### 4. Usage

After doing all the hard work, we will be able to access those variables in our stylesheets
from everywhere like this:

```css
.text {
  font-size: var(--font-size-sm);
  font-weight: var(--font-weight-sm);
}
```

## Want A less verbose approach ?

If you want a less verbose approach you have some solutions

### 1. HOC component

Create a Text component like what I described [here](https://portfolio-mili.vercel.app/gists/Text).

### 2. CSS classes

Create CSS classes in your stylesheets and use those classes in your markup to style texts.

```css
/* ... */
.tpg--sm {
  font-size: var(--font-size-sm);
  font-weight: var(--font-weight-sm);
}
.tpg--sm-b {
  font-size: var(--font-size-sm-b);
  font-weight: var(--font-weight-sm-b);
}
/* ... */
```

<small>
  ** If you find it hard to write it for all of the typography keys you need to
  automate it like what we did in the previous section (3. Adding values as CSS
  variables.)
</small>

<br />
<br />

To skip repetitive codes, you may want to automate the generation process with sass loops with something like this:

```css
/* _variables.scss */
$font-sizes: (
  h-xl: 32,
  h-lg: 24,
  h-md: 20,
  h-sm: 18,
  lg: 16,
  md: 13,
  sm: 11,
  xsm: 10,
  xxsm: 9,
);

/* utilities.scss */
.tpg {
  @each $tpg, $fs in $font-sizes {
    &--#{$tpg} {
      font-size: $fs;
    }
    &--#{$tpg}-b {
      font-size: $fs;
      font-weight: bold;
    }
  }
}
```

There are so many interesting things you can do with sass more than just simple features, you can see a bunch of them [here](https://portfolio-mili.vercel.app/gists/sass).
